# Анализ работ и сегментов (extract-jobs-from-reviews по Замесину)

**Дата:** 2026-05-14
**Источник:** `case-output/analyst/practicum-award-context.md` §2 (9 конкурентов в 3 слоях)
**Проанализировано отзывов/цитат:** 164 цитаты, разбитые по конкурентам Manus / Lindy / CrewAI / AutoGen / Devin / GigaChat / Yandex AI Studio / Just AI / ChatGPT-Claude / Big4 / фрилансеры
**Задача от менеджера:** применить 4-й промпт Замесина (extract-jobs-from-reviews) к собранным отзывам — превратить «сырьё цитат» в граф работ клиентов с верификацией Big Job и Core Job нашего сегмента.
**Ожидаемый инсайт:** какие реальные работы пытаются выполнить пользователи AI-агентов и традиционного консалтинга, и где сидит наш фокусный сегмент.
**Гипотеза менеджера (проверяем):**
- Big Job: «Принимать сильные стратегические решения в моменте и проводить через стейкхолдеров, чтобы развивать бизнес»
- Core Job: «Подготовить полный набор материалов решения бизнес-кейса»
**Выбранный метод:** AJTBD-извлечение из отзывов — единственный, который превращает теоретическую гипотезу в верифицированную из голоса клиента.

## 0. Шаг 1 — Сигналы из отзывов (выписка)

Перебор цитат по §4.2 промпта (глаголы желания / контекст / критерии / эмоции / проблемы / альтернативы).

### Сигналы из Manus AI

- «hallucinated progress — agent says 100% complete when reality is 37%» — **критерий успеха провален**: пользователь хочет *верифицируемый* результат, получает фейковый
- «scam, deleted my credits» (Trustpilot) — **негативная эмоция**: ощущение обмана; **триггер** — потеря денег без результата
- «$2 380 credits молча удалили при downgrade» — **проблема альтернативы**: непредсказуемость биллинга
- «14 failures in two weeks of testing» — **критерий успеха**: «работает с первого раза», провален
- «reliability degrades sharply with task length — long tasks break» — **контекст**: сложные многошаговые задачи; ожидание — агент справится сам; реальность — ломается
- «900 кредитов за одну complex задачу при балансе 19 900 на $199/мес» — **критерий**: «предсказуемая стоимость»
- «excellent researcher» (положительная цитата) — **глагол желания**: «хочу собрать research»; решает простые задачи

### Сигналы из Lindy AI

- «Expensive» — 42 упоминания (топ-1 жалоба) — **критерий**: «доступная цена за результат»
- «Hidden onboarding fee $1 500» — **проблема**: непрозрачность общей стоимости владения
- «Ease of use — 125 упоминаний» — **критерий успеха**: «без кода, без обучения»
- «Automation quality — handles nuance» — **глагол желания**: «хочу делегировать рутину с нюансами»
- Контекст положительных — email triage, scheduling, follow-ups; **триггер** — много рутинной коммуникации

### Сигналы из CrewAI

- «Debugging difficult — нельзя писать unit-tests» — **критерий**: «понимать, что внутри работает не так»
- «Latency issues при real-time нагрузке» — **критерий**: «отклик в адекватное время»
- «Rigid role-based structure — сложно для unconventional агентов» — **проблема**: «не подгоняется под мою задачу»
- «Start with CrewAI, then migrate to LangGraph at scale» — **сигнал альтернатив**: пользователь меняет инструмент по мере роста сложности
- **Контекст пользователя**: Python-разработчик, не бизнес-фаундер; глагол желания — «хочу построить *свою* систему», не «хочу получить готовый результат»

### Сигналы из AutoGen / Magentic-One

- Аналогично CrewAI: framework-сигналы, не end-user. Контекст — Microsoft-стек, .NET/Python разработчик.
- **Глагол желания:** «хочу собрать generalist multi-agent под свою задачу».
- Sigma бенчмарк-задач — **критерий**: «SOTA по бенчмаркам», но это разработческий критерий, не бизнес-пользователя.

### Сигналы из Devin

- «85% real-world fail rate» / «15% completion на сложных задачах» — **критерий успеха провален**: пользователь хочет «делегировал и забыл», получает 15%
- «14 fails из 20 на тестовых задачах» — **негативная эмоция**: обмануты ожидания «autonomous agent»
- Положительный контекст — **триггер**: «у меня бэклог хорошо описанных dev-задач, junior может сделать»; **глагол желания**: «делегировать junior-уровень»
- Снижение цены с $500/мес до $20/мес + $2.25/ACU — **сигнал критерия**: «доступная стоимость для одного разработчика»

### Сигналы из GigaChat Enterprise

- «70% рутинных операций берёт на себя» — **глагол желания корпоративного клиента**: «снимаю рутину с офисных сотрудников»
- **Контекст**: банк, госсектор, критическая инфраструктура; **триггер** — регламент уже описан, нужно автоматизировать
- **Проблема альтернатив**: не для МСП (нет публичного прайса, корпоративный канал продаж)

### Сигналы из Yandex AI Studio

- «Цена в 4× ниже прежней модели» (март 2026) — **критерий**: «доступная enterprise-цена»
- «Reasoning agents на DeepSeek-V3.2» — **глагол желания**: «хочу многошаговый reasoning по моим внутренним данным»
- **Контекст**: сравнение тендеров, анализ инцидентов, code work — *внутренние операционные задачи*, не стратегия

### Сигналы из Just AI

- Контекст-центры, корпоративные базы знаний — **глагол желания**: «хочу дать сотрудникам/клиентам семантический поиск»
- **Триггер**: уже есть большой корпоративный архив документов

### Сигналы из ChatGPT / Claude.ai (самые важные — это «дефолтная альтернатива»)

- «Спрошу у ChatGPT» / «Claude мне поможет» — **главная альтернатива** в голове фаундера МСП
- «Каждый запуск даёт разный результат» — **критерий провален**: «воспроизводимость»
- «Нет финмодели с формулами и sensitivity — только текстовая оценка» — **критерий**: «защищаемые формулами цифры»
- «Не дотренированы на эталонах кейс-чемпионатов — генерация шаблонна» — **негативная эмоция**: «получаю общую воду, не специфичный материал»
- «Один универсальный assistant, нет разделения ролей» — **проблема**: фаундер сам должен играть роль методолога, маркетолога, финансиста
- «Не верифицируют цифры через иерархию источников» — **критерий**: «цифры с источниками, которые выдержат вопросы стейкхолдеров»
- **Положительная сторона альтернативы**: $20/мес, мгновенный ответ, низкий порог входа — это и есть барьер для перехода к нам

### Сигналы из Big4-tier / «Яков и партнёры»

- «1-3 млн ₽ за discovery-проект» — **критерий провален для МСП**: цена недоступна
- **Положительный сигнал** (из их кейсов): «команда партнёров с верифицируемой репутацией» — это и есть **функция риск-снижения**, которую мы должны воспроизвести по-другому (через прозрачность методологии, не репутацию партнёров)
- **Контекст пользователя Big4**: дорогое стратегическое решение, M&A, выход на рынок — не МСП-фаундер

### Сигналы из фрилансеров (FL.ru / Профи.ру)

- «Презентация 2-30k ₽, финмодель 10-50k, research 15-45k» — **критерий**: «один артефакт — поштучно»
- «Полный пакет от одного фрилансера практически не встречается — нужно нанимать 5 разных человек, и никто не сшивает их работу в единую историю» — **главный сигнал работы**: пользователь хочет **сшитый, согласованный пакет**, а получает разрозненные артефакты
- **Триггер**: единая задача (питч, защита проекта, совет директоров), под которую нужны 4-5 разных артефактов с одной логикой

## 1. Сегменты по работам

Группировка по **похожести Big Job + Core Jobs**, а не по используемому инструменту (один и тот же человек может пробовать Manus, ChatGPT и фрилансера — но это один сегмент, если он решает одну работу).

---

### Сегмент 1. Фаундер / руководитель МСП в моменте стратегического решения  ← **НАШ ФОКУСНЫЙ**

**Big Job:**
> **Хочу принимать сильные обоснованные решения и проводить их через стейкхолдеров (инвесторы / совет / партнёры / команда), чтобы развивать бизнес и не упускать окно возможностей.**

*Сравнение с гипотезой менеджера:* формулировка менеджера верна по сути, но из отзывов яснее, что **«сила решения» = «способность защитить его перед стейкхолдером»**. То есть Big Job не просто «принять решение», а **«принять + защитить»** — это две неразрывные части одной работы. Это уточнение важно для волны маркетинга (Mot-Moment): триггер не «когда у меня вопрос», а «когда мне нужно убедить кого-то».

#### Core Job 1.1 — Подготовить полный пакет материалов решения бизнес-кейса под защиту  ← **ВЕРИФИЦИРОВАНА**

- **Когда**
   - Контекст: фаундер 5-50 человек, нанимает разные роли под задачу, бюджет на консалтинг ограничен
   - Триггер: предстоит питч инвестору / совет директоров / защита проекта внутри / переговоры с партнёром — нужно прийти с обоснованным предложением
   - Негативные эмоции ДО: «нанимаю 5 фрилансеров и сам сшиваю», «спросил ChatGPT — получил общую воду», «Big4 недоступен, фрилансер не вытянет стратегию», страх «выйти неподготовленным»
- **Хочу** получить методологически согласованный пакет (research + стратегия + финмодель + презентация) за обозримое время и стоимость, который выдержит вопросы стейкхолдера
   - Критерий успеха: артефакты не противоречат друг другу; цифры в финмодели подтверждены источниками; презентация рассказывает ту же историю, что и аналитика; могу защитить любую цифру
- **Чтобы**
   - Big Job: принять и защитить стратегическое решение
   - Позитивные эмоции ПОСЛЕ: «я готов», «у меня есть аргументы на любой вопрос», уверенность, профессиональное достоинство

#### Core Job 1.2 — Проверить свои гипотезы об рынке/конкурентах/клиентах до того, как принять решение

- **Когда**
   - Контекст: внутри есть гипотеза («думаю, нам надо в Азию» / «думаю, конкуренты не закроют сегмент X»)
   - Триггер: цена ошибки высокая (запуск продукта, наём команды, инвестиция)
   - Негативные эмоции ДО: «я не уверен, что данные у меня свежие», «боюсь, что я в пузыре собственных предположений»
- **Хочу** получить независимую проверку моих гипотез с цифрами и источниками
   - Критерий успеха: вижу, что цифры подтверждены 3+ независимыми источниками; вижу контр-аргументы, если они есть; могу пересмотреть гипотезу
- **Чтобы**
   - Big Job: принять решение, не упуская окно возможностей
   - Позитивные эмоции ПОСЛЕ: спокойствие, «я не пропустил очевидное»

#### Core Job 1.3 — Защитить цифры/выводы при противодействии стейкхолдера

- **Когда**
   - Контекст: на встрече с инвестором/советом стейкхолдер задаёт жёсткие вопросы — «откуда CAGR? почему 1100 ₽/час, а не 800? а если рынок не вырастет?»
   - Триггер: оппонент демонстрирует скепсис в реальном времени
   - Негативные эмоции ДО: страх «поплыву», страх «обнаружится, что ChatGPT галлюционировал»
- **Хочу** иметь готовые источники, sensitivity-сценарии и контр-аргументы под рукой
   - Критерий успеха: на каждую цифру есть источник; на каждое допущение есть пессимистичный / базовый / оптимистичный сценарий
- **Чтобы**
   - Big Job: провести решение через стейкхолдера
   - Позитивные эмоции ПОСЛЕ: «уверенно ответил», «оппонент признал силу аргументов»

**Типичные представители:**
- Тот, кто пробует Manus («hallucinated progress» — обжёгся на фейковом результате)
- Тот, кто платит Lindy $19.99 за рутину, но **для стратегии** идёт в ChatGPT
- Тот, кто нанимает 5 фрилансеров на FL.ru и сам сшивает презентацию + финмодель
- Тот, кто получил от Big4 квоту в 1.5 млн ₽ и закрыл вкладку

**Характерные цитаты (через отзывы конкурентов):**
- «Hallucinated progress — agent says 100% complete when reality is 37%» (Manus, Reddit) — *симптом*: «нет верифицируемости» — ключевая боль работы 1.3
- «Полный пакет от одного фрилансера практически не встречается» (фриланс-биржи) — *симптом*: рынок не предлагает Core Job 1.1
- «Каждый запуск ChatGPT даёт разный результат» — *симптом*: нет воспроизводимости артефактов, не выдерживает Core Job 1.3
- «Нет финмодели с формулами и sensitivity — только текстовая оценка» (ChatGPT/Claude) — *прямое опровержение Core Job 1.3*

**Проблемы с текущим решением:**
1. **ChatGPT/Claude** — даёт текст, не пакет; нет финмодели с формулами; шаблонный output; ноль воспроизводимости
2. **Manus / Devin / autonomous agents** — 85% failure rate без супервизии, фейковая прогресс-индикация, credit burn
3. **CrewAI / AutoGen / Yandex AI Studio** — это framework, не готовый результат; фаундер МСП не пишет Python
4. **Big4 / Яков и партнёры** — недоступная цена (1-3 млн ₽), долгие сроки
5. **Фрилансеры** — поштучные артефакты, никто не сшивает в единую историю; нужно нанимать 5 человек
6. **GigaChat / Just AI** — корпоративные платформы для процессной автоматизации, не для разовой стратегической работы

### Сегмент 2. Operations / business analyst в МСП — автоматизация рутины

**Big Job:**
> **Хочу освободить часы своего рабочего дня от повторяющейся коммуникации/координации, чтобы заниматься более ценной работой.**

#### Core Job 2.1 — Делегировать email triage / scheduling / follow-ups

- **Когда**
   - Контекст: operations-роль в малой команде, входящих сообщений много, контекст однотипный
   - Триггер: ежедневный поток рутинных писем съедает 2-3 часа
   - Негативные эмоции ДО: усталость, ощущение «я не двигаю компанию, я только реагирую»
- **Хочу** настроить AI-агента, который сам обработает входящие и сам поставит встречи
   - Критерий успеха: -50% времени на инбоксе, ошибки <5%, ease of use без кода
- **Чтобы**
   - Big Job: освободить часы для содержательной работы
   - Позитивные эмоции ПОСЛЕ: ощущение контроля, «я успеваю важное»

#### Core Job 2.2 — Автоматизировать обновление CRM/документов из переписки

- **Когда**
   - Контекст: каждый разговор с клиентом нужно занести в CRM/таблицу
   - Триггер: задача занимает 5-15 минут на разговор, накапливается
   - Негативные эмоции ДО: «я опять забыл занести лидов»
- **Хочу** агент, который слушает/читает контекст и сам обновляет систему
   - Критерий успеха: данные в CRM полные и свежие без моих ручных действий
- **Чтобы**
   - Big Job: освободить часы
   - Позитивные эмоции ПОСЛЕ: «всё под контролем, ничего не теряется»

**Типичные представители:** пользователи Lindy (125 mentions ease-of-use), AutoGen Studio low-code, отчасти Manus (для простой web-research рутины), GigaChat для корпоративного офиса.

**Характерные цитаты:**
- «Ease of use — главная сила» (Lindy, G2)
- «Automation quality handles nuance» (Lindy)
- «Excellent researcher» (Manus, для простых задач)

**Проблемы с текущим решением:**
- Lindy «expensive» (42 mentions) + hidden $1 500 onboarding
- Manus credit burn unpredictable
- Стоимость растёт быстрее линейно с объёмом

### Сегмент 3. Software engineer / технический специалист — делегирование dev-задач

**Big Job:**
> **Хочу увеличить выход своей команды разработки, делегируя junior-уровень задач AI-агенту, чтобы фокусироваться на сложной архитектуре.**

#### Core Job 3.1 — Делегировать junior-уровень dev-задач из backlog

- **Когда**
   - Контекст: senior разработчик / tech lead, в бэклоге много хорошо описанных мелких задач (баг-фиксы, простые фичи)
   - Триггер: бэклог растёт, junior-разработчиков нет/нет бюджета
   - Негативные эмоции ДО: «я застрял на мелочах, не двигаюсь по архитектуре»
- **Хочу** запускать агента на конкретную задачу и получать PR на ревью
   - Критерий успеха: PR работает с первой-второй попытки, экономит часы
- **Чтобы**
   - Big Job: увеличить выход команды на дорогую работу
   - Позитивные эмоции ПОСЛЕ: «я двигаюсь по тому, что действительно важно»

#### Core Job 3.2 — Построить свою multi-agent систему под специфический бизнес-процесс

- **Когда**
   - Контекст: внутренний инженер enterprise, есть конкретный бизнес-процесс под автоматизацию
   - Триггер: руководство хочет AI-агента, готовые решения не подходят
   - Негативные эмоции ДО: усталость от proof-of-concept, «не хочу платить за то, что собирается из open-source»
- **Хочу** удобный фреймворк (CrewAI, AutoGen, Yandex AI Studio) с гибкими ролями и observability
   - Критерий успеха: можно прототипировать за день, видна логика, есть unit-tests, latency приемлемая
- **Чтобы**
   - Big Job: увеличить выход команды
   - Позитивные эмоции ПОСЛЕ: «у меня есть рабочий прототип, могу масштабировать»

**Типичные представители:** пользователи Devin, CrewAI, AutoGen, Yandex AI Studio (Agent Atelier), Just AI Flow.

**Характерные цитаты:**
- «Devin 15% completion на сложных задачах» — *симптом*: пользователь обжигается на ожидании автономности
- «Start with CrewAI, then migrate to LangGraph at scale» (Reddit) — *симптом*: пользователь меняет инструмент с ростом сложности
- «Debugging difficult — нельзя писать unit-tests» (CrewAI)

**Проблемы с текущим решением:**
- Devin: 85% failure rate на сложных задачах, ACU-биллинг непредсказуем
- CrewAI: debugging, rigid roles, latency
- AutoGen: maintenance mode → миграция на Microsoft Agent Framework

### Сегмент 4. Корпоративный enterprise-клиент — процессная автоматизация

**Big Job:**
> **Хочу системно снять рутинные операции с офисных сотрудников при сохранении безопасности и регуляторного контроля, чтобы оптимизировать OPEX.**

#### Core Job 4.1 — Автоматизировать обработку входящих заявок / документов по корпоративным регламентам

- **Когда**
   - Контекст: банк / госкорпорация / крупный B2B; регламент уже формализован
   - Триггер: руководство ставит цель -X% FTE на функцию
   - Негативные эмоции ДО: давление KPI, ответственность за compliance
- **Хочу** платформу, на которой можно настроить агента под регламент с контролем доступа и аудит-логом
   - Критерий успеха: 70% рутинных операций берёт на себя; SOC2 / 152-ФЗ соблюдены; on-premise возможен
- **Чтобы**
   - Big Job: оптимизировать OPEX
   - Позитивные эмоции ПОСЛЕ: «выполнили KPI, не нарушили compliance»

#### Core Job 4.2 — Дать сотрудникам/клиентам семантический поиск по корпоративной базе знаний

- **Когда**
   - Контекст: есть тысячи документов (регламенты, инструкции, kb)
   - Триггер: сотрудники тратят часы на поиск, обращаются в helpdesk
   - Негативные эмоции ДО: «знание есть, но недоступно»
- **Хочу** RAG-агент, который отвечает по корпоративной базе с цитированием источников
   - Критерий успеха: ответы корректные, источники указаны, никаких утечек наружу
- **Чтобы**
   - Big Job: оптимизировать OPEX, повысить скорость работы сотрудников
   - Позитивные эмоции ПОСЛЕ: «инвестиция в knowledge оправдалась»

**Типичные представители:** клиенты GigaChat Enterprise, Yandex AI Studio (крупный сегмент), Just AI Agent Platform / Jay Knowledge Hub.

**Характерные цитаты:**
- «До 70% рутинных операций берёт на себя» (GigaChat позиционирование)
- «Цена в 4× ниже прежней модели» (Yandex AI Studio, март 2026)

**Проблемы с текущим решением:**
- Нет публичного прайса — продажи долгие
- Требуется внутренняя команда для настройки
- Не решает разовую стратегическую работу — это процессная платформа

### Сегмент 5. Knowledge worker / самозанятый специалист — точечный ответ на вопрос

**Big Job:**
> **Хочу быстро получить помощь по конкретному вопросу в моменте, чтобы продолжить свою работу.**

#### Core Job 5.1 — Получить быстрый ответ / структуру / черновик на конкретный запрос

- **Когда**
   - Контекст: маркетолог, аналитик, junior-PM, студент; рабочая задача с дедлайном
   - Триггер: «надо за 30 минут собрать структуру презентации / draft email / список конкурентов»
   - Негативные эмоции ДО: ступор перед белым листом, нехватка времени
- **Хочу** ввести промпт и получить адекватный draft, который можно доработать
   - Критерий успеха: ответ за минуты, без кода, бесплатно или $20/мес
- **Чтобы**
   - Big Job: продолжить работу
   - Позитивные эмоции ПОСЛЕ: «не застрял, успеваю»

**Типичные представители:** массовый пользователь ChatGPT / Claude.ai (800M+).

**Характерные цитаты:**
- «Спрошу у ChatGPT» — массовый дефолт
- Claude Artifacts для генерации HTML-черновиков

**Проблемы с текущим решением:**
- Шаблонный output, недостаточный для серьёзных задач
- Нет воспроизводимости
- Нет специализации под отрасль / задачу

## 4. Выводы и инсайты для менеджера

### Инсайт 1. Big Job менеджера верна, но требует уточнения формулировки

**Факт:** Гипотеза менеджера — «Принимать сильные стратегические решения в моменте и проводить через стейкхолдеров, чтобы развивать бизнес». Сигналы из отзывов (Manus hallucinated progress, ChatGPT «каждый запуск разный», фрилансеры «никто не сшивает», Big4 недоступен) подтверждают эту работу — но раскрывают, что **«принять» и «защитить» это две неразрывные части одной работы**, не последовательные.

**Что это значит:** Mot-Moment персоны должен быть не «когда у меня вопрос», а **«когда мне нужно принять решение И защитить его перед стейкхолдером»** — этот двойной триггер делает Core Job 1.1 + 1.3 неразрывными.

**Возможное использование:** Маркетолог волны 2 должен строить персону на тригерах типа «питч в среду», «совет в пятницу», «совещание с партнёром через неделю» — а не на абстрактном «я хочу AI».

---

### Инсайт 2. Core Job менеджера верна — и она единственная не закрытая на рынке

**Факт:** «Подготовить полный набор материалов решения бизнес-кейса» = Core Job 1.1 в нашем графе. Из 9 проанализированных конкурентов **ни один не закрывает её целостно**:
- ChatGPT/Claude дают текст, но не пакет с финмоделью и согласованной историей
- Manus/Devin падают на сложных задачах (85% fail rate)
- CrewAI/AutoGen/Yandex AI Studio — это framework, не продукт
- GigaChat/Just AI — другая работа (процессная автоматизация)
- Big4 закрывает по сути, но за 1-3 млн ₽
- Фрилансеры закрывают поштучно, но «никто не сшивает в единую историю»

**Что это значит:** Это и есть «голубой океан» нашей дифференциации. Все остальные сегменты уже имеют закрывающий продукт, а Сегмент 1 — нет.

**Возможное использование:** В слайде «зачем мы нужны» нет смысла перечислять много преимуществ — достаточно одного тезиса «**никто не делает сшитый пакет под защиту**», и под ним — матрица 9 конкурентов × Core Job 1.1 (все «нет»).

---

### Инсайт 3. Главная негативная эмоция нашего сегмента — страх «обнаружится фейк»

**Факт:** Самые острые цитаты из отзывов — про **верифицируемость**: «hallucinated progress», «каждый запуск разный», «нет финмодели с формулами», «$2 380 credits молча удалили». Это не про цену и не про скорость — это про **доверие к результату**.

**Что это значит:** Главный страх нашего ЦА — не «дорого» и не «долго», а **«я приду на питч с фейковыми цифрами и опозорюсь»**. Это объясняет, почему Big4 жив несмотря на цену — клиенты платят за «снижение риска опозориться».

**Возможное использование:** В концепции и в питче `/case-team` ключевой месседж не «дёшево» и не «быстро», а **«каждая цифра защищена источником, каждая методология воспроизводима»**. Это и есть наша подмена «партнёрской репутации Big4» — мы продаём не репутацию, а **прозрачность методологии**.

---

### Инсайт 4. ChatGPT — главный когнитивный конкурент, но он же — наш onboarding-фуннел

**Факт:** «Спрошу у ChatGPT» — массовый дефолт для всех сегментов. Сегмент 1 (наш) тоже сначала идёт туда, упирается в «шаблонный output» и «нет финмодели», и только потом ищет альтернативу.

**Что это значит:** Мы не должны конкурировать с ChatGPT «фронтально» — наша работа начинается там, **где ChatGPT заканчивается** (когда фаундер понял, что нужен пакет под защиту, не draft).

**Возможное использование:** В позиционировании — фраза типа «**когда ChatGPT недостаточно для защиты решения**». Это снимает когнитивный конфликт «зачем мне платить вам, если есть ChatGPT за $20» — мы про *другую* задачу.

---

### Инсайт 5. Сегмент 1 имеет ограниченную частотность (2-6 раз/год) — это влияет на бизнес-модель

**Факт:** Фаундер МСП принимает крупное стратегическое решение не каждый день. Цикл — раунды, советы, M&A, выход на рынок — 2-6 раз в год.

**Что это значит:** Это **не SaaS-подписка по природе** — это либо проектная модель, либо подписка с очень редким использованием (как Big4 ритейнер). Подписочная модель сработает только если мы расширим Core Jobs (например, добавим Job 1.2 «проверка гипотез» как более частую — это уже до 1-2 раз в месяц).

**Возможное использование:** В финмодели — частотность работы 2-6 раз/год даёт ограничение на «средний чек ÷ частотность» = LTV. Это аргумент **в пользу проектной цены, не подписки** для текущего этапа. Финансист должен учесть это в Unit Economics.

---

### Инсайт 6. Противоречие в данных: Big4 жив, хотя в 100× дороже ChatGPT — потому что закрывает РИСК, не работу

**Факт:** Big4 за 1-3 млн ₽ существует параллельно с ChatGPT за $20/мес — оба обслуживают «стратегические решения», но клиенты разные. Big4 не падает в цене до уровня AI.

**Что это значит:** Цена ≠ единственный критерий. Big4 продаёт **«мы возьмём на себя репутационный риск»** — это и есть невыполняемая ChatGPT работа. Наш сегмент 1 хочет того же риск-снижения, но не может заплатить 1-3 млн ₽.

**Возможное использование:** Наша модель — **«снижение риска за 1/30 цены Big4 через методологическую прозрачность вместо партнёрской репутации»**. Это сильный месседж для слайда сравнения.

---

### Инсайт 7. Сюрприз — у Lindy 125 mentions «ease of use», но 42 mentions «expensive». ROI плохой при простой работе

**Факт:** Lindy выигрывает на «легко настроить», но проигрывает на «дорого за то, что получаю». Скрытая onboarding fee $1 500 — повторяющаяся жалоба.

**Что это значит:** Для нашего сегмента ease of use **не главный критерий** — главный критерий «выдержит ли защиту». Не нужно копировать стратегию Lindy «no-code и красивый UI». Это другая работа.

**Возможное использование:** В нашей концепции **не делать акцент на «легко»** — это уведёт в сегмент 2 (operations). Делать акцент на **«методологично, защищаемо, согласовано»** — это удерживает фокус на сегменте 1.

## 5. Самопроверка

- **Гипотеза менеджера по Big Job:** **подтверждена с уточнением формулировки** (см. Инсайт 1). Финальная формулировка: «Принимать решения и проводить их через стейкхолдеров» — двойная неразрывная работа.
- **Гипотеза менеджера по Core Job 1.1:** **полностью подтверждена** (см. Инсайт 2) — это единственная незакрытая на рынке работа из всех сегментов.
- **Заданный инсайт закрыт:** да — менеджер получил верифицированную Big Job + Core Jobs + ранжированные сегменты для маркетинга волны 2.
- **Противоречия в данных:** Big4 vs ChatGPT (Инсайт 6) — это не ошибка данных, а реальный конфликт двух разных работ; зафиксирован отдельно.
- **Чего не хватает (передаю менеджеру / финансисту):**
  1. **Финансисту:** учесть частотность Core Job 1 (2-6 раз/год) в Unit Economics — это влияет на выбор «проектная модель vs подписка»
  2. **Маркетологу волны 2:** ключевые триггеры Mot-Moment — «питч в среду / совет в пятницу / переговоры с партнёром», а не абстрактное «нужна стратегия»
  3. **Менеджеру:** уточнить, использовать ли Job 1.2 (проверка гипотез) как «второй продукт» для увеличения частотности — это решение менеджера, не аналитика
