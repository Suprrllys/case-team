## Резюме

`/case-team` — открытая мультиагентная система на базе Claude Code, которая готовит **полный набор материалов решения бизнес-кейса под защиту перед стейкхолдерами** за 4 часа реального времени, для пользователя Claude — без дополнительной оплаты (в стоимости подписки на нейросеть, которая уже оплачена для других задач).

**На выходе — физические рабочие файлы**, не текст в чате: `.pptx` презентация с навигацией, `.md` финмодель с sensitivity, аналитика с прослеживаемыми источниками, Decision Log с обоснованием каждой развилки. Это снимает главный страх клиента — «обнаружится фейк под вопросами стейкхолдера».

**Эта заявка собрана этой же командой `/case-team`** за день до дедлайна — мета-демо тезиса «один продуктовый менеджер + команда из 5 ИИ-агентов = эффективная единица решения бизнес-кейсов». Все материалы открыты для жюри в репозитории.

## 1. Проблема

В России **266 тыс компаний** малого и среднего бизнеса (источник: [ФНС, единый реестр МСП](https://rmsp.nalog.ru/) · [Ведомости 25.12.2025](https://www.vedomosti.ru/economics/articles/2025/12/25/1166484-chislo-subektov-msp-uvelichilos)) принимают стратегические решения **2-6 раз в год**: питч инвестору, реакция на изменение рынка, запуск нового продукта, защита перед советом директоров. На каждое решение нужен сшитый пакет аналитики, финмодели, концепции, позиционирования и презентации.

Существующие способы получить этот пакет имеют конкретные минусы:

| Способ | Стоимость 1 кейса | Главный минус |
|---|---|---|
| Делать самому | 5-6 дней личного времени | Слабее в одной из 5 ролей; история не сшивается |
| In-house команда из 5 человек | ~99 тыс ₽ + 700 тыс ₽/мес ФОТ | Неподъёмно для МСП |
| Big4-tier агентство (или «Яков и Партнёры») | 350 тыс — 1.5 млн ₽ | Дорого, долго, нет вовлечённости |
| Фрилансеры (5 человек) | 50-150 тыс ₽ | Полный пакет не встречается — нет куратора смысла |
| ChatGPT / один универсальный AI-чат | стоимость подписки | Текст, не пакет; нет финмодели с формулами; каждая генерация — разный результат |

После ухода MBB (McKinsey / BCG / Bain) из РФ в 2022 г. ([HSE Daily](https://daily.hse.ru/post/novye-igroki-rynki-perspektivy-kak-vyglyadit-rynok-upravlencheskogo-konsaltinga-v-2023-godu), [Fless.pro](https://fless.pro/big3-in-russia-after-2022-sbs-consulting)) средний бизнес остался в полном вакууме: стратегический консалтинг есть только у крупных корпораций.

**Объём недополученного спроса в РФ:** 18-80 млрд ₽/год — деньги, которые рынок не может потратить на стратегические задачи из-за барьера стоимости консалтинга.

## 2. Работа клиента (AJTBD)

В описании целевой аудитории используется формат сегментации по работам — методология **Advanced Jobs To Be Done** (Christensen, Moesta), активно популяризируемая в российском продакт-сообществе Иваном Замесиным. Не «мотивация / момент потребления / барьеры» в свободной форме, а **граф работ клиента**:

**Главная работа клиента (верхний уровень):** принимать сильные обоснованные решения **и** проводить их через стейкхолдеров — инвесторов, совет директоров, партнёров, команду — чтобы развивать бизнес.

«Принять» и «защитить» — это неразрывная двойная работа: одного без другого не бывает. Это уточнение получено из реальных отзывов клиентов конкурентов (см. [граф работ из 164 цитат](https://github.com/Suprrllys/case-team/blob/main/solution/analyst/jobs-and-segments-analysis.md)).

**Что выполняет `/case-team` полностью** (это и есть точка покупки): подготовить полный набор материалов решения бизнес-кейса под защиту.

В набор входит:
- Презентация `.pptx` — с навигацией, pre-flight check, готова к показу
- Финмодель `.md/.xlsx` — с формулами и sensitivity ±20%
- Аналитика — рынок, конкуренты, ЦА с активными URL-источниками
- Концепция и позиционирование (AJTBD-формулировки)
- Decision Log — обоснование каждой развилки

**Что остаётся за человеком** (граница ответственности):
- Принять финальное go/no-go решение
- Защитить решение лично перед стейкхолдерами (живая коммуникация)
- Внедрить стратегию операционно
- Договориться с конкретными партнёрами / нанять людей

Чёткая граница ответственности — сознательный выбор: мы не обещаем «AI принимает стратегические решения». Жюри-практики знают, что AI не принимает решений на стратегическом уровне; это анти-фактор доверия. Мы обещаем **полную подготовку для принятия решения и его защиты** — это конкретно, измеримо, верифицируемо.

**Измеримые критерии успеха** (как клиент понимает, что работа выполнена хорошо):
- **Скорость:** 4 часа реального времени работы менеджера (vs 5-6 дней у команды из 5 человек)
- **Прослеживаемость:** 100% цифр в финальных материалах имеют активную URL-ссылку на первичный источник
- **Защищаемость в Q&A:** на каждую цифру в основной презентации — кликабельная ссылка
- **Воспроизводимость:** один запрос `/case-team` → структурно идентичный пакет из 5 файлов
- **Низкий порог входа:** 1 команда `/case-team` в Claude Code; 0 шагов настройки MCP / subagents / pipeline

**Главная негативная эмоция клиента до выполнения работы** — не «дорого», а **«страх обнаружится фейк под вопросами стейкхолдера»**. Это объясняет, почему Big4 жив при 1-3 млн ₽ за один питч при наличии ChatGPT за подписку: клиенты Big4 покупают защиту от позора через репутацию партнёра. Наше решение функционально замещает «защиту через репутацию» прозрачностью методологии — каждая цифра прослеживается до источника в Decision Log.

## 3. Решение — архитектура из 5 ролей

`/case-team` — мультиагентная система с явным разделением ответственности. Один менеджер-человек запускает команду одной слэш-командой, дальше агенты работают параллельно по структурированному workflow.

```
                  ┌──────────────────┐
                  │  case-manager    │   ◄── куратор смысла
                  │  PM-оркестратор  │
                  └────┬─────────────┘
                       │ Agent tool (задача с контекстом)
                       ▼
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
   ┌─────────┐   ┌──────────┐   ┌──────────┐
   │analyst  │   │financier │   │marketer  │
   │research │   │финмодель │   │AJTBD     │
   └────┬────┘   └────┬─────┘   └─────┬────┘
        │ результат-.md в свою папку  │
        └──────────────┼──────────────┘
                       │ (cross-agent через blockers/, всегда через менеджера)
                       ▼
                ┌─────────────┐
                │case-designer│ ◄── конвертер формата
                │YAML.md→.pptx│
                └─────────────┘
```

**Четыре архитектурных решения, которых нет в существующих multi-agent системах** ([Manus AI](https://felloai.com/manus-ai-pricing/), [Lindy](https://www.lindy.ai/), [CrewAI](https://crewai.com/), [AutoGen](https://www.microsoft.com/en-us/research/articles/magentic-one-a-generalist-multi-agent-system-for-solving-complex-tasks/), [Devin](https://devin.ai/), [GigaChat Enterprise](https://habr.com/ru/companies/sberbank/news/1005990/), [Yandex AI Studio](https://aistudio.yandex.ru/ru), [Just AI](https://agentplatform.just-ai.com/)):

**1. Менеджер-агент — куратор смысла внутри команды.** Решает, что класть на слайды; специалисты не пишут сразу в `.pptx`. Это сознательный ответ на 85% failure rate автономных агентов: Devin показывает 13.86% completion на сложных задачах ([независимый тест на SWE-Bench](https://www.openaitoolshub.org/en/blog/devin-ai-review)). Curatorship — недостающий слой в существующих multi-agent системах.

**2. На выходе — физические рабочие файлы**, не текст в чате. Когда жюри / инвестор / совет директоров открывает результат — он видит `.xlsx` с формулами, `.pptx` готовый к показу, `.md` Decision Log с источниками. Это критическое отличие от ChatGPT (текст без артефактов) и от фрилансеров (артефакты есть, но никто не сшивает в единую историю).

**3. Дотренировка на двух реальных кейс-чемпионатах** — решение для **Axenix** на [Cup Moscow 2024](https://www.cup-moscow.ru/) (стратегический B2G кейс, авиахаб Звартноц) и решение для **Askona** на [Inno Case Hack 2025](https://innocasehack.ru/) (стартап-маркетинговый кейс, A-Sleep). Из них извлечено **26 методологических логик** в каталоге команды. Дотренировка не на финальных результатах, а на **рабочих артефактах** — черновики, Decision Log, отвергнутые идеи. Это даёт агентам понимание **как думать**, а не просто «как выглядит хороший слайд».

**4. Иерархия источников в каждом правиле агента.** Каждое правило агента ссылается на свой уровень источника:
- **Эталон** (Break the Case 2024-2025) → конкретные приёмы с обоснованием из реального кейса
- **Бизнес-классика** (Christensen JTBD, Osterwalder Value Proposition, MoSCoW, AS-IS/TO-BE, AJTBD-инструменты Замесина) → валидные рамки методологии
- **Эвристика менеджера** → честно помеченные приёмы без эмпирической базы

Это снижает галлюцинации в стратегических цифрах — каждое утверждение можно проследить до источника.

### 5 ролей команды

| Агент | Что делает | Главный методологический инструмент |
|---|---|---|
| `case-manager` | Концепция, Decision Log, 6-шаговый алгоритм отбора инсайтов на слайды, делегирование | Эталон Break the Case + классика стратегического консалтинга |
| `case-analyst` | Research рынка, конкурентов, ЦА, извлечение работ из реальных отзывов | Каталог исследовательских фреймворков + 4 AJTBD-промпта Замесина |
| `case-financier` | Финмодель, NPV, sensitivity, TAM, юнит-экономика | Универсальная инструкция финмодели (5 блоков Input × 11 вертикалей) |
| `case-marketer` | AJTBD-сегментация, Mot-Moment персона, каскад целей, позиционирование, слоганы | Motivation-Moment Persona + Value Exchange + Barrier-Concept Matching |
| `case-designer` | Конвертирует YAML.md → `.pptx` через python-pptx + 4 проверки качества | Универсальная инструкция презентации + скилл `claude-design` (14 фазовых процедур) |

Полная архитектура и материалы — в открытом репозитории: [github.com/Suprrllys/case-team](https://github.com/Suprrllys/case-team).

## 4. Экономика

В подаче экономики мы различаем **внутренний расход на работу решения** (себестоимость) и **цену для клиента**. Это два разных уровня финмодели, и их смешение — типичная методологическая ошибка.

### Цена для пользователя Claude

**$0 дополнительно**. Подписка Claude ($20-100/мес) уже оплачена пользователем для других задач: написание кода, редактура, research. `/case-team` — это открытый skill для Claude Code, который устанавливается одной командой и расходует токены той же подписки. Нет отдельной транзакции для пользователя — это категориальный сдвиг, не price war.

### Внутренний расход на 1 кейс (для прозрачности)

| Сравнение базы | Стоимость 1 кейса, ₽ |
|---|---|
| **Команда агентов** (3.5 ч менеджера × 1500 ₽ + Claude tokens ~$77.5 × 73.34 ₽/$ = 5 680 ₽) | **~10 930** |
| In-house команда из 5 человек (90 чел.-часов × 1100 ₽/час средневзвешенная middle) | 99 000 |
| Big4-tier агентство (×4 multiplier за presale, маржу, ритейнеры) | 396 000 |

**Sensitivity ±20% по 3 параметрам** (ставка консультанта, время менеджера, расход токенов): экономия устойчива в диапазоне **84-98%** vs in-house. Подробные расчёты с источниками — [solution/financier/unit-economics.md](https://github.com/Suprrllys/case-team/blob/main/solution/financier/unit-economics.md).

### Эффект в год для типичного B2B-клиента

При частотности 1 кейс в квартал (типичный B2B-режим — 4 крупных стратегических решения в год):
- vs in-house: **~350 тыс ₽/год** экономии
- vs Big4-tier: **~1.5 млн ₽/год** экономии

### Бенчмарки экономии AI в knowledge work

Наши 84-98% выше публичных бенчмарков потому что мы работаем в **substitute-режиме** (заменяем 4 из 5 ролей одной командой агентов), а большинство публичных бенчмарков измеряют **assist-режим**:

- [Anthropic](https://www.anthropic.com/research/estimating-productivity-gains) — 80-81% медиана экономии времени на задачах с Claude (исследование на 100k диалогов)
- [Klarna](https://www.klarna.com/international/press/klarna-ai-assistant-handles-two-thirds-of-customer-service-chats-in-its-first-month/) — $40 млн/год экономии, эквивалент 700 FTE, 82% сокращение resolution time
- [GitHub Copilot](https://github.blog/news-insights/research/research-quantifying-github-copilots-impact-on-developer-productivity-and-happiness/) — 55% быстрее, peer-reviewed контролируемый эксперимент (P=0.0017)

### TAM в РФ

- **~266 тыс компаний МСП** в фильтре «малый + средний + крупный бизнес с регулярными стратегическими задачами» (источники: [ФНС реестр МСП](https://rmsp.nalog.ru/), [Корпорация МСП](https://xn--90aifddrld7a.xn--p1ai/news/country/chislennost-msp-v-rossii-obnovila-istoricheskiy-maksimum-i-prevysila-6-8-mln-predpriyatiy/), [Гарант](https://www.garant.ru/news/1964227/))
- **Объём недополученного спроса:** 18-80 млрд ₽/год — деньги, которые рынок не может потратить на стратегические задачи

Расчёт TAM и sensitivity — [solution/analyst/practicum-award-context.md §1.4](https://github.com/Suprrllys/case-team/blob/main/solution/analyst/practicum-award-context.md).

## 5. Что меняется относительно существующих способов

| Способ сейчас | Что меняется с `/case-team` |
|---|---|
| **Делать самому** (5-6 дней) | 4 часа реального времени; качество в 5 ролях одновременно, недостижимое одному человеку |
| **In-house команда из 5** | Не нужен 1-2-месячный найм; пакет за стоимость подписки на нейросеть |
| **Big4-tier агентство** (350k-1.5М ₽, 4-8 недель) | 4 часа вместо недель; снижение риска через прозрачность методологии вместо репутации партнёра |
| **Фрилансеры с биржи** | Куратор смысла внутри (менеджер-агент) — единая методология, согласованные артефакты |
| **ChatGPT / один AI-чат** | Получаешь `.xlsx` и `.pptx` с формулами, не текст в чате; на каждую цифру — активный URL-источник |

### Низкий порог входа

| Барьер клиента | Как решён |
|---|---|
| Технический («настраивать MCP, subagents, pipeline») | 1 команда `/case-team` в Claude Code; 0 шагов настройки multi-agent pipeline |
| Доверия («AI не справится со стратегией и опозорит на питче») | Дотренировка на 2 реальных кейс-чемпионатах; 100% цифр имеют URL-источник |
| Формата («получу ли то, что нужно показать стейкхолдеру») | Гарантированные 5 файлов на выходе: `.xlsx` финмодель, `.pptx` презентация, `SOLUTION.md`, `MANAGER-LOG.md`, research |
| Обучаемости («надо изучать промпт-инжиниринг») | 0 команд изучения промпт-инжиниринга; задача ставится естественным языком |
| **Цены/риска** | Снят полностью через категориальный сдвиг — нет отдельной транзакции для пользователя Claude |

## 6. Технологический стек

| Компонент | Что | Обоснование |
|---|---|---|
| **Модель** | Claude Opus 4.7 (1M context) | Длинный контекст для многошаговых кейсов с цепочкой источников; state-of-art reasoning для синтеза разнородных материалов |
| **Архитектура** | Claude Code subagents | Изолированные роли с собственными промптами, инструментами, папками |
| **Интеграции** | MCP-серверы (Notion, Google Drive), WebSearch / WebFetch | Доступ к внешним источникам без копирования вручную |
| **База знаний** | Локальные эталоны (Cup Moscow + Inno) + универсальные инструкции (финмодель, презентация) + каталог фреймворков | Дотренировка на реальных рабочих артефактах, не синтетике |
| **Методология** | AJTBD (Christensen / Moesta) + AJTBD-инструменты Замесина (4 промпта) + Osterwalder Value Proposition + MoSCoW / AS-IS-TO-BE | Иерархия источников с явными ссылками в правилах каждого агента |
| **Дизайн-качество** | Скилл `claude-design` (реверс-инженерный системный промпт Anthropic Claude Design, MIT) + 14 фазовых процедур (ai-slop-check, accessibility-audit, hierarchy-rhythm-review, polish-pass) | Контроль качества финального артефакта |
| **Сборка `.pptx`** | python-pptx + pre-flight overflow check + macOS PowerPoint blessing | Воспроизводимая программная сборка |
