## 2. Что обязательно сделать пользователю (macOS PowerPoint blessing)

> **Это известная особенность python-pptx + macOS PowerPoint.** Без этого шага PowerPoint
> будет ругаться диалогом «обнаружена проблема с содержимым» при открытии файла.
> Описано в Универсальной инструкции презентации § 3.

**Шаги:**

1. Открыть файл `practicum-award-2026-case-team.pptx` двойным кликом в Finder
   (PowerPoint запустится автоматически)
2. Появится диалог «PowerPoint обнаружил проблему с содержимым» —
   **нажать «Восстановить»**
3. PowerPoint восстановит файл и откроет его (контент полностью сохранится)
4. Меню **File → Save As → PowerPoint Presentation (.pptx)**
   - В Recent Locations выбрать ту же папку `case-output/designer/`
   - В имени файла указать `practicum-award-2026-case-team-FINAL.pptx`
   - **Эту копию загружать в форму подачи** — она уже «благословлённая» PowerPoint
     и откроется без диалога у жюри

**Альтернатива (если PowerPoint недоступен):** открыть в Keynote → File → Export To
→ PDF → загружать в форму подачи PDF.

## 3. Шрифт — что произойдёт на машине жюри

В builder'е явно указан `YS Text` (body) и `YS Display` (heading) — это
фирменные шрифты Яндекса. Они **не установлены по умолчанию** на машинах
жюри (и, скорее всего, не установлены на вашей machine разработки тоже).

**Что произойдёт при показе:**

- macOS PowerPoint увидит указанный typeface `YS Text` в `<a:latin/>` каждого
  run, поищет его в системе. Не найдёт.
- Заменит на ближайший системный sans-serif (обычно **Arial** на macOS, **Calibri**
  на Windows).

**Это безопасно:** компоновка не развалится, потому что Arial/Calibri имеют
близкие метрики; pre-flight overflow check заложен с safety margin.

**Если хочется родного бренда** — установить шрифты YS Text / YS Display
заранее, скачав с [https://github.com/yandex-fonts](https://github.com/yandex-fonts)
или из брендбука Яндекс. Но не критично для подачи.

**Чисто для надёжности можно после blessing-шага в PowerPoint:**
- Меню Format → Replace Fonts → заменить `YS Text` на `Inter` (если установлен)
  или `Arial`. То же для `YS Display`.
- Save As.

```
✓ saved: practicum-award-2026-case-team.pptx
```

Проверяет:
- Все `<p:sp>` имеют `y + h ≤ 7.5"` (нижняя граница слайда 16:9)

## 5. PDF-экспорт — ручной шаг

LibreOffice (`soffice`) **не установлен** на этой machine — автоматический
рендер `.pptx → .pdf` или `.pptx → .png` недоступен.

**Варианты:**

| Способ | Команда / шаги |
|---|---|
| **PowerPoint** (после blessing) | File → Export → Create PDF/XPS Document |
| **Keynote** | File → Export To → PDF |
| **LibreOffice** (если установить) | `brew install --cask libreoffice && soffice --headless --convert-to pdf practicum-award-2026-case-team.pptx` |

Рекомендую сразу же создать PDF после blessing — на случай, если форма
конкурса попросит PDF.

## 6. 4 проверки качества (claude-design) — статус

Согласно YAML-контракту требуется запустить 4 параллельных проверки:

| Проверка | Статус | Комментарий |
|---|---|---|
| `ai-slop-check` | **пропущена** | Требует PNG-рендеров слайдов; нет LibreOffice / автоматического экспорта PowerPoint |
| `accessibility-audit` | **пропущена** | То же |
| `hierarchy-rhythm-review` | **пропущена** | То же |
| `polish-pass` | **пропущена** | То же |

**Что было сделано взамен (статические проверки в коде builder'а):**

| Критерий | Статус |
|---|---|
| Не больше 3 цветов в композиции (primary + bg + один акцент) | ✓ соблюдено |
| Не больше 7±2 смысловых блоков на слайд | ✓ соблюдено |
| Не больше 4 колонок в таблице | ✓ соблюдено (max 3 cols на A0) |
| Заголовок = ТЕЗИС, не тема | ✓ из YAML, дословно |
| Иконки — векторные, эмодзи запрещены | ✓ нет эмодзи; используются круги/полоски/числа как иконки-якори |
| Pre-flight overflow check | ✓ пройден |
| Кликабельные URL на источниках A0 | ✓ 71 hyperlink через `run.hyperlink.address` |
| Контраст ≥ 4.5:1 | **визуально проверено**: чёрный текст на белом, белый текст на чёрном — обе пары соответствуют WCAG AA. Жёлтый primary на чёрном — 14.5:1 (отлично). Жёлтый primary на белом — 1.5:1 (только для декора, **не для основного текста**) |

**Рекомендация:** после blessing-шага и экспорта в PDF — пройтись глазами по
слайдам 1 раз вслух, проверить тайминг (10 мин лимит).

## 9. Известные ограничения

1. **PowerPoint warning «обнаружена проблема с содержимым»** —
   неизбежен у файлов python-pptx на macOS. Решение: blessing-шаг (§2).
   Это **не баг файла**, это особенность macOS PowerPoint.

2. **Шрифты YS Text/YS Display, скорее всего, не установлены** —
   будут заменены на Arial. См. §3. Не критично.

3. **Картинка фото Сергея отсутствует** — будет placeholder. См. §8.

4. **PNG-рендеры слайдов не получить автоматически** — нет LibreOffice.
   4 проверки качества claude-design выполнены в редуцированном виде (по
   статическим критериям в коде builder'а). Рекомендую визуальную ревизию
   после blessing-шага.
